## Context

走查现状（基于真实代码）：

- **顶栏**（`Layout.tsx` header）：左侧 `t('headerTitle')` + `v{version}` + 更新 Badge；右侧 accel(⚡) 按钮 + 帮助菜单。左侧品牌与侧栏 logo/`brandName` 重复，中段大片留白。
- **散落的状态**：侧栏底部 `taskRunning` pill（跳 `/recent-tasks`）与 `downloadPill`（跳 `/engines`）；`update-status` / `addon-fallback` / `message` 经 toast；日志在 `LogDialog`。无「一处看全」的活动汇聚。
- **单运行器事实**：`getTaskStatus` 仅返回 `running | idle`，即同一时刻只有一个转写 / 翻译 / 合并任务在跑（模型下载可并行）。所以活动中枢是「实时状态中心」而非大型渲染队列。
- **命令 / 键盘**：`cmdk@^1.1.1` 与 `ui/command.tsx` 已在仓库，但仅 `ProviderForm` 当组合框用；全局热键仅 `mod+,` 与 `?`。
- **工程**：`getWorkItems` 返回 work items（名称 + 经 `getWorkItemTarget` 的 deep-link）；校对任务自成体系。

约束：**内容 / 数据 / IPC / 引擎 · 翻译 · 任务 · 校对行为冻结**；不引入新三方依赖；字体沿用 system 栈。

## Goals / Non-Goals

**Goals:**

- 把顶栏升格为**上下文工具条**，作为活动中枢与命令面板的共享画布，消除品牌冗余与中段真空。
- 让「正在发生的一切」（运行中任务 + 下载 + 最近完成 / 失败）在**一处** popover 可见可控。
- 提供键盘优先的全局 **Cmd+K** 跳转 / 最近工程 / 动作。
- 去除页级与分区标题的同名回声；为数据界面补 skeleton 与统一空态。

**Non-Goals:**

- 不做多任务并发队列 / 调度（后端是单运行器）。
- 不做逐页上下文命令（v2）、不做可停靠 / 可调分栏布局（D5，另案）。
- 不做统一编辑台（D3，独立大赌注）。
- 不改 `/recent-tasks` 作为全量档案的职责（活动面板只管实时 + 最近少量）。

## Decisions

### 决策①：活动面板用 popover（非右侧 sheet）

锚定顶栏状态按钮的 **popover**。活动中枢是「瞥一眼」的高频轻量动作，popover 出现 / 消失快、不抢占内容区；sheet 更重、与内容竞争横向空间，更适合编辑型长任务。面板底部「查看全部」跳 `/recent-tasks` 承接深度浏览。

- 备选：右侧 sheet（否，太重）；独立路由（否，与 recent-tasks 职责重叠）。

### 决策②：版本 / 更新提示落在活动面板页脚 + 按钮更新点

顶栏不再背静态「品牌 + 版本」块（正是要去的冗余）。版本号与「有新版本 vX」属于**应用状态**，归入活动面板页脚这一「应用状态」位；同时当 `update-status=available` 时，在顶栏活动按钮上加一个**小圆点**，使更新提示无需点击即可被察觉——满足「保留版本因为有更新提示」。

- 备选：顶栏保留 mono 小字版本（否，重新引入冗余）；仅藏帮助菜单（否，弱化更新提示）。

### 决策③：accel(⚡) 保留为常驻独立 chip

GPU / 加速是用户重视的**常驻环境信号**且是直达引擎加速区的快捷入口，折叠进面板会埋没它。保留为右侧状态簇里独立 chip，与活动按钮**并置**。

- 备选：并入活动面板（否，降低可见性与可达性）。

### 决策④：标题去重——顶栏承载子页上下文，枢纽页保留 PageHeader

顶栏「页面上下文」主要服务**子工作面**（`tasks/[type]`、校对编辑态）的「返回 + 当前名」；枢纽页继续保留 `PageHeader`（描述 / 操作有价值）。仅移除**纯重复**的分区标题回声（如 translation 页与页头同名的首个分区标题）。不整体废除 PageHeader。

- 与 `page-layout-rhythm` 关系：该 spec 规定标题两层写法（`label-caps` / `text-sm font-semibold`），本次只是**移除一处同名回声**，不改其写法契约，故不产生 delta。
- 备选：顶栏页名彻底替代所有 PageHeader（否，丢失描述与页级操作位、改动面过大）。

### 决策⑤：命令面板 v1 范围 = 跳转 + 最近工程 + 全局动作

复用 `ui/command.tsx`，`mod+k` 唤起。三组：

- **跳转**：`NAV_ITEMS` 六项 + `/recent-tasks`。
- **最近工程**：`getWorkItems` → 经 `getWorkItemTarget` 直达。
- **动作**：复用既有 handler——新建转写 / 翻译 / 转写+翻译任务、切换主题、检查更新、查看日志、打开引导、FAQ、GitHub、展开 / 折叠侧栏。
  逐页上下文动作留 v2。
- 备选：一上来做上下文感知动作（否，范围与风险膨胀）。

### 决策⑥：补一小批「跳转」导航快捷键并登记进 ShortcutsHelpDialog

在 `useHotkeys` 增量注册保守、低冲突的导航快捷键（`mod+k` 开面板；以及一组「跳转到」导航键），全部在 `ShortcutsHelpDialog` 列出以可发现。避免与浏览器 / 系统及现有输入态冲突（沿用 `allowInInput` 语义）。

### 决策⑦：Skeleton 基元基于现有 token

新增 `ui/skeleton.tsx`（`bg-muted` + `tailwindcss-animate` 的 `animate-pulse`），无新依赖。数据界面在「加载中且无缓存数据」时渲染与最终布局同构的骨架，避免「居中转圈 + 跳变」。空态统一走现有 `EmptyState`。

### 决策⑧：外壳以「色阶台阶」而非硬分割线分区（L 型 chrome 面）

原先侧栏 / 顶栏 / 内容三者同为 `bg-background`，仅靠三条 1px `--border`（侧栏 `border-r`、logo `border-b`、顶栏 `border-b`）切分，且 logo 与顶栏的两条 `border-b` 在左上角交汇成**硬十字**，读感生硬。沿用既有「三层中性面」哲学，新增语义面 `--chrome`（深 `0 0% 9%`、浅 `0 0% 97%`），令**侧栏 + 顶栏**合为一块安静面板（L 型 chrome），内容区维持画布 `--background`（深 7% / 浅 99%），内容卡片仍是 `--card`（深 11% / 浅 100%）——形成 `画布 → chrome → 卡片 → 浮层` 单调抬升。移除上述三条硬线：分区改由 chrome 与画布之间的**色阶台阶**界定，左上角转角随之无缝（logo 融入侧栏、侧栏融入顶栏）。为补回「嵌定感」/边界清晰度，仅在内容区 `<main>` 上加 `border-l border-t border-border/60` 的**单层发丝线**——因 `<main>` 起于顶栏下方，该线只勾勒 chrome↔画布的**内 L 边**（顶栏下沿 + 侧栏右沿的内容段），顶栏与侧栏仍为同色无缝 chrome，不复现网格盒，也不在转角形成十字；`/60` 较旧实心 `--border` 明显更柔。

- 零新依赖（仅 1 个 token + 1 处 tailwind 颜色注册 + 1 条发丝线）；纯外壳视觉，未触内容、IPC、功能。
- 备选：浮动内容卡（圆角 + 阴影 + 内边距，否——动高度 / 滚动管线，风险中）；仅把 border 降不透明度（否——仍是线条思路，未根除「网格感」与转角十字）。

## Risks / Trade-offs

- **顶栏是全局外壳，改动牵一发动全身** → 分阶段：先立「上下文工具条」骨架（保持现有功能可用），再挂活动中枢、再挂命令面板；每阶段在 Electron 窗口逐页核对。
- **活动面板与现有侧栏 pill / toast 职责重叠** → 明确分工：活动面板=可展开的「全量实时」；侧栏 pill 可保留为「正在进行」的极简常驻指示（或在面板落地后精简），避免双重信息源打架；本设计先共存，落地阶段评估是否收口 pill。
- **命令面板动作触达过多状态** → v1 只接已存在且无副作用风险的 handler；破坏性动作（如删除工程）不进 v1。
- **i18n 漂移** → 新文案集中在 `common`（外壳）；逐页核对中英标签宽度（顶栏在英文较长）。
- **标题去重可能误伤** → 仅删「与页头同名」的回声，逐页人工确认语义不丢。

## Migration Plan

1. 立顶栏「上下文工具条」骨架（去品牌冗余、留版本 / 更新、加占位状态簇与 Cmd+K 入口），功能与现状等价。
2. 落地活动中枢 popover（消费既有事件），版本 / 更新移入页脚 + 按钮更新点。
3. 落地命令面板（跳转 / 最近工程 / 动作）+ 导航快捷键 + 速查登记。
4. 标题去重 + skeleton / 空态铺设。
5. 逐页运行态核对（深 / 浅、中 / 英、各页返回 · 动作 · 状态）。

- 回滚：各阶段独立提交；顶栏骨架与旧实现行为等价，可单点回退某阶段。
