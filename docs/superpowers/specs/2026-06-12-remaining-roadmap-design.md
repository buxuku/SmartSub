## 7. B17 whisper addon native abort 接入(等待 addon.node 交付)

**触发条件:用户的新 addon.node(含 AbortController 支持)构建完成并交付。** 接口已确认:

```js
const ac = new AbortController();
const promise = whisperAsync({ ...params, signal: ac.signal });
ac.abort(); // 任意时刻取消
const result = await promise;
```

### 7.1 接入范围

1. **接入点**:`main/helpers/subtitleGenerator.ts` 的 `whisperAsync(whisperParams)` 调用——whisperParams 增加 `signal`,AbortController 由任务取消上下文(B1 已建的 `taskContext.ts`)派生,与 ffmpeg 注册表(`audioProcessor.ts`)同模式管理。
2. **取消语义**:取消任务 → 立即 abort 正在运行的转写,文件状态回退为可重跑,临时音频清理,UI spinner 停止——P0#2 验收「点取消 5 秒内真正停止」从「队列级」升级为「推理级」。
3. **暂停语义(批次设计时定)**,两个选项:
   - (a) 维持现状:停止派发,当前文件跑完(无算力浪费,但「暂停」不即时);
   - (b) abort 当前转写并回退该文件,恢复时重跑(即时停,但 whisper 无断点续传,长文件重跑成本高)。
   - 倾向:取消=立即 abort;暂停先维持 (a) 并保持文案明示「暂停=不再开始新文件」,若用户反馈强烈再升级 (b)。
4. **错误区分**:abort 引发的退出(AbortError)不得标记文件为 failed,与真实转写错误分流。
5. **addon 分发**:新 addon.node 经现有 addonDownloader/addonVersions 管线分发,版本号与兼容性标记更新。

### 7.2 验收标准(预设)

1. 转写进行中点取消,5 秒内 spinner 停止、进程 CPU/GPU 占用回落;
2. 取消后立即重跑同文件,结果正确(abort 后内部状态/显存释放干净,连续 abort+重跑 10 次稳定);
3. abort 的文件不显示「失败」,显示「已取消」;
4. 多平台 addon 变体(metal/coreml/cpu/vulkan/cuda)行为一致;
5. 三项门禁全绿。

### 7.3 附注:python-engine 线(本轮不接入)

`feat/python-engine-poc` 的 faster-whisper sidecar 为另一条独立验证线,用户已明确本分支及本轮需求不考虑接入。预分析备忘留存:CTranslate2 模型格式与 ggml 不兼容、CTranslate2 无 Metal 支持(Mac 仅 CPU int8)、PyInstaller bundle 体积数百 MB。
