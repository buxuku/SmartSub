# 批次 10 设计：术语统一与 i18n 治理

对应 UX 报告 4.4（文案与术语系统）、4.5（国际化质量）；Phase 3 验收标准 ③（zh/en key 对等 CI 绿）④（术语表全量替换无「听写/识别」残留）。

## 背景与探索结论

- 术语漂移（zh）：「听写」6 处、「识别」7 处、「转写」21 处三词混用；「显卡加速」10 处 vs「GPU 加速」11 处；侧边栏「视频合字幕」与启动台「合成到视频」不一致；错别字「源始字幕设置」。
- en 侧已基本统一 Transcribe，少量 Recognition 残留。
- i18n 兜底：266 处 `t('key') || '中文兜底'`，已验证 key 全部存在且 zh/en key 完全对等（差异=0）→ 可机械移除。
- 死键：保守扫描 219 个疑似，其中 `provider.*`、`launchpad card.*` 等为动态拼接 key 误报；真死键预计 150+（settings/subtitleMerge/translateControl 的旧版残留）。
- 硬编码英文：`LogDialog.tsx` DialogDescription 纯英文。
- namespace 重组（拆 proofread/editor）：本批次不做（机械量大、风险中、用户无感知；报告 11.4 允许 i18n 类延后）。

## 术语表（T1 执行口径）

| 旧                            | 新           | 说明                                                                      |
| ----------------------------- | ------------ | ------------------------------------------------------------------------- |
| 听写 / 识别（语音转文字语境） | 转写         | en 统一 Transcribe；「自动识别」语言选项（autoRecognition）语境不同，保留 |
| 显卡加速                      | GPU 加速     | 长文案首次出现可括注「（显卡）」                                          |
| 视频合字幕                    | 合成到视频   | 仅功能入口名；「烧录/Burn-in」保留在硬字幕输出模式技术语境                |
| 源始                          | 原始         | 错别字                                                                    |
| 配置条「字幕样式」            | 「输出内容」 | 标签与值（仅输出翻译字幕等）语义对齐                                      |

注意：`tasks.json stage.transcribe` 阶段链 zh 是「听写」→ 改「转写」；onboarding 多处长文案逐条人工改写保持语句通顺。

## 任务拆分

1. **T1 术语表全量替换**：zh+en locale 全量扫描替换 + 人工复核每处语境；组件内中文常量同步（如有）。
2. **T2 死键清理 + 硬编码 i18n 化**：先收集代码中全部动态 `t(\`...\`)` 拼接前缀形成豁免清单，复核后删除真死键（zh/en 同步删）；LogDialog 描述补 key。
3. **T3 移除兜底**：脚本替换 `t('key') || '...'` → `t('key')`（含带插值参数形态），人工抽查 diff。
4. **T4 校验脚本**：`scripts/check-i18n.mjs`——检查 zh/en key 对等、检测新增 `t() || ` 兜底模式；`package.json` 挂 `check:i18n`。
5. **T5 门禁**：renderer TSC 非测试 0 错误；main 95 基线；`check:i18n` 绿；提交并交接。

## 风险

- 动态 key 误删：以豁免清单 + 人工复核控制；删除后跑 `check:i18n` 与全文 grep 双确认。
- 术语替换破坏插值/标点：逐条 diff 复核而非盲目 sed。
