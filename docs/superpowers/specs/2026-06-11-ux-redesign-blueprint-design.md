> 关联文档：`RESOURCE_HUB_REDESIGN_PLAN.md`（资源中心详细方案，本蓝图吸收其骨架并升级两处）

## 1. 理念与背景

SmartSub 定位为**专业的字幕创作台，小白用户也能轻松上手**。当前形态的核心问题：

| 问题             | 现状                                                                               |
| ---------------- | ---------------------------------------------------------------------------------- |
| 首页信息密度过高 | 三种任务类型 + 全部配置项挤在左侧大面板，加重抗拒心理与认知负担                    |
| 工序割裂         | 生成 / 校对 / 合成是三个孤立页面，数据互不相通，用户需自己发现「生成完可以去校对」 |
| 供给侧分散       | 模型、翻译服务、GPU 加速分散在三处，新用户不知道「先要下模型」                     |
| 模型管理体验差   | 推荐不显眼、卡片留白多、分类不直观                                                 |
| 无新手引导       | 不下模型、不配翻译服务，软件基本不可用，但没有任何引导                             |
| 任务不持久       | 任务列表存内存，重启即丢；专业工作台需要任务历史与产物追溯                         |
| 专业词汇过多     | VAD、CUDA、量化、maxContext 等直接暴露给用户                                       |

## 2. 设计原则（所有后续裁决的依据）

1. **任务优先于配置**：默认值即最佳实践；配置渐进披露（常用 → 内联配置条，其余 → 高级抽屉）。
2. **概念在需要时解释**：不回避「语音模型 / 翻译服务 / 显卡加速」三个核心概念（它们是软件可用的前提），但首次接触必配人话解释；其余术语一律收进高级区并配说明。
3. **工序自然衔接**：每个完成态都递出下一步（生成 → 校对 → 合成 → 打开文件夹）。
4. **状态可见**：就绪度（模型/翻译/加速）全局可感知；任务进度一眼可读。
5. **克制的精致**（视觉风格 A）：层次靠间距与排版，不靠颜色轰炸；动效只用于状态反馈。

## 3. 信息架构

### 3.1 导航（侧边栏 6 → 5 项）

```
🏠 任务（启动台）      ← home 重构
✏️ 字幕校对
🎬 合成到视频
📦 资源中心            ← 新增，吸收 模型管理 / 翻译管理 / 设置内 GPU 卡片
⚙️ 设置
```

- 校对 / 合成「双入口」：启动台卡片是门厅，侧边栏是地图，互不取代。
- 旧路由 `/modelsControl`、`/translateControl` 保留薄重定向（沿用资源中心草案）。

### 3.2 路由

| 路由                                                | 内容                                                         |
| --------------------------------------------------- | ------------------------------------------------------------ |
| `/{locale}/home`                                    | 启动台（新首页）                                             |
| `/{locale}/tasks/[type]`                            | 任务页 ×3：`generate-translate` / `generate` / `translate`   |
| `/{locale}/resources?tab=...`                       | 资源中心 4 Tab：overview / models / providers / acceleration |

## 4. 启动台（新首页）

线框：`launchpad-taskpage.html` ①

- **任务卡片 ×5**，文案用「输入 → 输出」语言去专业化：
  - 视频 → 双语字幕（识别人声并翻译成目标语言，一步到位）
  - 视频 → 原文字幕（只识别人声，不翻译）
  - 翻译已有字幕（把 SRT 等字幕文件翻译成其它语言）
  - 校对字幕（逐句检查修改，支持视频对照与 AI 润色）
  - 合成到视频（把字幕烧录进画面，永久显示）
- **就绪横幅**（未就绪时）：顶部黄色横幅，一句人话解释 + 主 CTA（如「下载推荐模型 (1.5GB)」），不遮挡不强制。
- **卡片就绪标记**：需要模型的卡片右上角「需要模型」小标；点击未就绪卡片引导去下载而不是死卡。
- **拖文件到卡片** = 选类型 + 导入一步完成。
- **最近任务区**：最近 5 条任务摘要（文件名、类型、状态、时间），点击直达对应任务页/工作区——缓解老用户「多一步点击」的成本。
- 精致感来源：大间距、统一图标语言、卡片 hover 微提升、克制的彩色图标底。

## 5. 任务页（×3）

线框：`launchpad-taskpage.html` ②

- **取消左侧大配置面板**，改为顶部**内联配置条**：标签 + 当前值，点开 popover 选择。只放 4-5 个常用项：
  - 生成类：语音模型 / 视频语言 / 翻译成 / 翻译服务 / 字幕样式（双语等）
  - 仅生成：语音模型 / 视频语言 / 字幕格式
  - 仅翻译：源语言 / 翻译成 / 翻译服务 / 字幕样式
- **高级选项抽屉**（右侧 Sheet）：提示词、输出格式、源/译字幕保存设置、并发数、重试次数、VAD（自设置页迁入）、上下文长度等，每项配人话说明。
- **任务行重设计**（替代现在的三列图标）：
  - 阶段指示（提取 › 识别 › 翻译，当前阶段加粗）+ 单一进度条 + 百分比/预估剩余时间
  - 完成行内动作：[校对] [合成] [📂 打开所在文件夹]
  - 错误行：红色状态 + 人话错误 + 重试动作
- **完成时刻**（peak-end rule）：全部完成时出绿色成果横幅——「N 个视频已完成」+ 产物文件清单 + [去校对] [合成到视频] [打开文件夹]。
- **日志面板**：底部可折叠终端风面板，默认收起，实时滚动，可关闭。
- 配置自动记住（沿用现有 userConfig 机制），页头提示「配置会自动记住」。
- 空状态：保留大拖放区 + 说明；消灭「服务商未加载时整面板空白」这类静默失败。

## 6. 工序衔接与任务持久化

线框：`pipeline-flow.html`

### 6.1 衔接动线（快捷通道，不是强制流程）

```
生成/翻译完成 ──行内[校对]，自动带视频+字幕──▶ 校对编辑器
校对标记完成 ──完成横幅[合成到视频]，自动带校对稿──▶ 合成页（视频+字幕已预填）
合成完成 ──[打开成品文件夹]──▶ 交付
```

全程零重选文件；每道工序仍可从启动台独立进入。

### 6.2 合成页预填

`subtitleMerge` 支持接收入参（视频路径 + 字幕路径），从衔接入口进入时跳过 FileSelector 手动选择。

### 6.3 统一任务历史（WorkItem）

```
WorkItem {
  id, type(双语/原文/翻译/校对/合成),
  源文件, artifacts[](产物字幕/视频),
  status(排队/进行中/已完成/出错/已中断),
  各阶段进度, 配置快照, createdAt/finishedAt
}
```

- 存储 electron-store，取代「内存任务列表 + 独立校对任务」两套互不相通的数据。
- 重启恢复：运行中任务重启后标「已中断」，一键重跑；已完成任务保留产物链接，随时 [校对][合成][📂]。
- 启动台「最近任务」、任务页列表、校对历史共用此模型。
- 顺手正型：现 `IFiles` 的 `extractAudio?: boolean` 等类型与实现（实为状态字符串）脱节，在 WorkItem 建模时一并解决。

### 6.4 校对编辑器路由化

现状是首页内嵌状态切换（`setProofreadFile`）。收敛为独立路由页，从任务行 / 校对页进入的是同一个编辑器，支持返回。

### 6.5 系统通知

任务全部完成且应用在后台时发系统通知。

## 7. 资源中心

**沿用 `RESOURCE_HUB_REDESIGN_PLAN.md` 骨架**（统一 `/resources` 入口、四 Tab、侧边栏收编、旧路由重定向、IPC 零改动、i18n namespace 保留策略），升级两处：

### 7.1 模型 Tab 重设计（不再「零改动平移」）

线框：`resources-onboarding.html` ①

| 痛点       | 设计                                                                                                                                 |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| 推荐不显眼 | 顶部「为这台电脑推荐」hero 区：大字模型名 + 推荐依据（内存/加速状态）+ 一键下载大按钮                                                |
| 卡片留白多 | 紧凑行式布局：名称 / 人话描述 / 速度·准确度评分点 / 大小 / 状态（下载按钮）同一行                                                    |
| 分类不直观 | 按用途三档：🚀 快速档（tiny/base）· ⚖️ 均衡档（small/medium）· 🎯 高精度档（large 系），每档一句适用人群说明；真实模型名保留为副信息 |
| 变体杂乱   | 量化版 →「轻量版」、纯英文版，折叠进「展开更多」                                                                                     |

顶部工具行保留：下载源切换 / 导入本地模型 / 存储路径 / 「只看已安装」筛选。

### 7.2 全景 Tab 与新手引导统一

全景 = 常驻就绪状态总览（readiness dashboard），新手引导 = 首次流程。两者共用同一套就绪状态推导逻辑（模型已装？翻译已配？加速何种后端？），文案与状态色一致。

## 8. 新手引导（首启向导，3 步，随时可跳过）

线框：`resources-onboarding.html` ②

1. **它是怎么工作的**：一张原理图——视频 → 【语音模型】听写 → 【翻译服务】翻译 → 字幕文件，顺带解释显卡加速（可选增强）。三个核心概念一次讲清。
2. **下载一个语音模型**：按内存自动选中推荐档（也可换成「先快速试试」的 tiny）；下载在后台进行，不阻塞下一步。
3. **可选增强（强烈建议）**：
   - 翻译服务：说明「要翻译字幕才需要」，免费起步推荐 DeepLX / Ollama 本地 / 火山引擎，[去配置] 深链接资源中心。
   - 显卡加速：展示检测结果一键开启。平台文案——Windows：「NVIDIA 显卡加速（CUDA）/ 通用显卡加速（Vulkan）」；macOS：「Apple 芯片加速（CoreML / Metal）」。

- 完成后落启动台；跳过的事项由启动台就绪横幅 + 资源中心全景 Tab 持续兜底（同一套状态逻辑，两条路都能达成可用状态）。
- 重新打开：顶栏「？」帮助菜单。
- 触发条件：首次启动（无已装模型且无引导完成标记）。

## 9. 文案与术语策略

- 三个核心概念保留但必配解释：语音模型（把声音听写成文字）、翻译服务（把文字翻译成目标语言）、显卡加速（让听写快几倍）。
- 术语翻译表（首批，实施时全量审计 zh/en）：

| 现状                         | 改为                                             |
| ---------------------------- | ------------------------------------------------ |
| VAD                          | 人声检测                                         |
| CUDA                         | NVIDIA 显卡加速                                  |
| Vulkan                       | 通用显卡加速                                     |
| CoreML / Metal               | Apple 芯片加速                                   |
| 量化版                       | 轻量版（精度略降，更省内存）                     |
| maxContext / 上下文长度      | 高级项 + 人话说明                                |
| 生成并翻译 / 仅生成 / 仅翻译 | 视频 → 双语字幕 / 视频 → 原文字幕 / 翻译已有字幕 |

- 错误信息人话化并给解决动作（如下载失败 → 建议切换下载源）。
- 任务类型卡片统一「输入 → 输出」句式。

## 10. 视觉语言（风格 A：克制中性）

参考：`visual-style.html` A 方案。

- 延续 shadcn slate 基因；亮/暗双主题保留，默认浅色。
- 间距 4/8 栅格；圆角统一 8/12px；阴影只用于浮层与 hover 提升。
- 状态色规范：绿=完成/就绪，黄=待处理/中断，蓝=进行中，红=错误。
- 进度条统一组件（任务行 / 模型下载 / 加速包下载共用）。
- 动效仅用于状态反馈（hover 提升、面板展开、横幅出现），150-200ms ease。
- 图标统一 lucide；任务类型用克制的彩色图标底做区分。

## 11. 全局其它优化

1. **设置页瘦身**：GPU 卡片迁资源中心（草案已含）；VAD 迁任务页高级抽屉；whisper 命令模板等极客项收进「高级」折叠区；保留语言、更新、路径、导入导出、危险区。
2. **顶栏帮助菜单（？）**：重开新手引导、查看日志、文档 / GitHub。
3. **空状态体系化**：所有页面空状态都有解释 + 下一步 CTA（含任务页模型/服务商下拉为空 → 深链接资源中心，即草案 T6）。
4. **技术债顺手修**（随对应期实施）：
   - `useIpcCommunication` 监听器泄漏（cleanup 返回空函数）
   - 侧边栏菜单配置化（数组驱动，替代 6 份样板）
   - 清理未接线的参数系统 Demo 组件（~3000 行）、旧 Guide 残骸、废弃 i18n key
   - 两套字幕编辑 hook（`useSubtitles` / `useStandaloneSubtitles`）在编辑器路由化时合并
5. **i18n**：新增 `launchpad` / `tasks` / `onboarding` / `resources` namespace；存量 namespace 沿用草案的保留策略；zh/en 同步。

## 13. 风险与缓解

| 风险                             | 缓解                                                            |
| -------------------------------- | --------------------------------------------------------------- |
| 老用户肌肉记忆（首页多一步点击） | 启动台「最近任务」直达；配置自动记住；旧路由重定向              |
| P3 与 P5 数据模型衔接            | P3 的任务列表组件以 WorkItem 接口为契约开发，P5 只换存储不动 UI |
| 重构周期长、主干漂移             | 每期独立可发版、原子提交；P1 已有成熟草案先行降险               |
| 向导被跳过导致不可用             | 就绪横幅 + 全景 Tab 兜底，两条路径都能达成可用状态              |
| 双语维护成本                     | 每期验收含 zh/en 完整性检查                                     |

## 14. 非目标（本蓝图不做）

- 不改转写/翻译底层引擎与 IPC 协议（除 WorkItem 持久化所需的任务存储层）
- 不做在线模型市场（仍是内置静态清单）
- 不引入翻译服务商 `enabled` 开关（沿用草案 §5.3 决策）
- 不做遥测/数据埋点
- 不改品牌视觉（Logo、应用名）
